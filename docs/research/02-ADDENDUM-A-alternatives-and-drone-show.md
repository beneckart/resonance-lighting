# DOCUMENT 2 of 5 — ADDENDUM A: Alternatives & Drone-Show Protocols

Every major design choice deep-checked — alternatives, why rejected, honest limitations.

**A1 — Controller transport.** Chosen: WebSocket (binary tweak frames + JSON commits), WebRTC held as the upgrade. Rejected: OSC/UDP (browsers can't send UDP → forces a native app, kills PWA iteration); sACN/Art-Net direct to fixtures (association storms, the brownout failure, ~10× power); BLE (range/throughput). Adopted as resilience: a ~100-line Art-Net listener on the cortex so any pro iPad lighting app can drive the tree in an emergency. Limits: BM 2.4GHz contested (AP ~30–50 m); WS is TCP (one dropped packet stalls briefly — hence the upgrade path).

**A2 — App platform.** Chosen: React Three Fiber PWA. Rejected: native Swift (app-store friction kills on-playa iteration), Unity (heavy, licensing), Chromatik/LX desktop (laptop, wired-era — concepts borrowed, not runtime), TouchDesigner (laptop overkill). Limits: iOS may evict unused PWA storage (re-install pre-playa); throttle the twin when backgrounded.

**A3 — Mesh protocol.** Chosen: ESP-NOW, control-plane only, broadcast + synced clocks + heartbeat-echo repair (PDR ≥97% measured). Rejected: WiFi mesh (association fragility at low SOC, 10× idle power), BLE mesh (100s-ms latency), Zigbee/Thread (new radios on all 150), LoRa-as-control (seconds latency). Limits (honest): no broadcast ACK — a fixture asleep through all 3 repeats waits for its next heartbeat to be repaired (worst-case ~2s convergence); 250-byte payload cap; **single-hop** — fixtures must hear the master directly (crown antenna covers the canopy; root-shadowed fixtures are the risk → RF-walk the built tree).

**A4 — Central brain.** Chosen: Jetson Orin Nano Super 8GB DC-direct + NX slot + phone spare. Rejected: Mac mini M4 (best perf/W but **AC-only, violates the DC rule**); phone-as-primary (2–6W unbeatable but 16GB caps at 8B, throttling, charge-at-heat battery risk → perfect spare, wrong primary); Pi 5 (8B at 2–4 tok/s CPU, conversationally dead); RK3588 (cheapest 16–32GB DC but rkllm tooling jank → fallback); Ryzen AI Max 395 128GB (70–120B, ~300–450 Wh/night → 2027 path); clustered boards (exo/llama.cpp-RPC — network latency where the voice pipeline can't afford it, N× failure modes). Limits: 8GB ceiling until NX; freeze the JetPack image pre-playa. **Bench-verify (don't ship on faith):** DC window under inference load, NX-on-carrier thermal, iPad PWA mic, DJI enumeration, DJI+LoRa+mesh sweep, RF-walk.

**A5 — Sensing.** Chosen: PIR ×150 + mmWave sentinels + BME280 + chime IMUs. Rejected: mmWave everywhere (power), cameras (privacy/culture/dust — absolute no), ultrasonic (dust/wind noise), ToF/lidar (power/dust on optics), CSI-only (experimental → P2 spike). Limit: **PIR can't see a motionless person** (door counters + sentinels carry stillness; engine encodes "quiet ≠ empty"); ±30% zone estimates (gauge, not count).

**A6 — Voice.** Chosen: DJI-in-conch + shell exciter + 8B. Rejected: wired lav (more reliable, less magical — kept as fallback in the bundle), trunk mic array (worse intimacy/SNR — the conch IS the noise solution), cloud LLM (latency/censorship/ban + no internet), bigger model (waits for NX/2027). Limit: exciter loses to an art car at 50m (accepted — intimate, not a sound stage).

**A7 — Camp bridge.** Chosen: Meshtastic + BM Internet. Rejected: custom LoRa firmware (months), ESP-NOW LR (congested 2.4GHz), dish link (power/aim/mast), Starlink (20–40W kills the budget). Limit: kilobits — digests + commands only; must run a private PSK channel.

**A8 — Show storage.** Chosen: cues + schedule on cortex, mirrored to master flash. Rejected xLights/FPP FSEQ *as a streamed pixel format* — but its compile-then-play model is right (→ A9).

**A9 — DRONE-SHOW PROTOCOLS + the Show Compiler (P1.5).** Drone shows pre-upload each drone's full program, verify on the ground, then showtime is a *clock, not a stream* — a drone that loses radio finishes perfectly. We are stationary drones. The **Show Compiler:** locking a schedule compiles per-fixture show files (keyframe tracks `[t, pattern_id, params, transition]`, ~4–16 KB/fixture) referencing firmware pattern engines; distributes them across the afternoon (SOC-gated, each fixture echoes the show-hash in its heartbeat); at showtime the master sends only time sync → jitterless, and **total master failure mid-show is invisible**. Jam-mode params ride as an override layer that decays back. Readiness gates (Monitor go/no-go): ≥95% fleet, SOC floor, clock sync ≤±10ms, hash verified. Abort → compiled ambient track; blackout → GPIO4; beacon preempts. **GPS timebase ($10):** drift-free week-long clock + true sunrise/sunset for dusk/dawn automation. Not taken: RTK positioning, per-agent trajectories, proprietary C2 radios.

**Verdict:** design holds. Two upgrades adopted — the Art-Net shim and the Show Compiler + GPS. One honesty promoted: PIR's motionless blind-spot is now an explicit occupancy-engine constraint.
